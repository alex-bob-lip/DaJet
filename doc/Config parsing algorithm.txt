# Алгоритм чтения метаданных

1. Получаем данные файла DBNames:

SELECT BinaryData FROM Params WHERE FileName = N'DBNames'

{e496a657-001e-474e-a364-6336965b9034,"Reference",36},
{33405315-3023-41e2-a11c-8965c269ce17,"Fld",37},
{cab9f251-2656-4e97-950d-f2d4588ddc3a,"VT",45},
{cab9f251-2656-4e97-950d-f2d4588ddc3a,"LineNo",46},

2. Получаем данные файла для основного объекта метаданных.

SELECT BinaryData FROM Config WHERE FileName = @FileName ORDER BY PartNo ASC

@FileName - внутренний идентификатор, UUID объекта метаданных, например, справочника, который был получен из файла DBNames.

3. Получаем наименование объекта метаданных и его синоним.
4. Если это справочник, то определяем является ли он подчинённым. Если да, то получаем информацию о справочниках владельцах.
5. Ищем табличные части и получаем их реквизиты.
6. Получаем реквизиты самого объекта метаданных.
